セットアップ & 操作ガイド ― 役職者向け意思決定支援ツール

✅ 本ドキュメントはWindows・Mac両対応のセットアップ手順と操作方法を説明します。

1. COVEとは

COVEは、役職者の意思決定を多角的な視点からサポートするAIツールです。完全ローカル動作を基本とし、機密性の高い情報を外部に送信することなく利用できます。必要に応じてクラウドAIとの切り替えも可能です。

  • 🎙️ 録音・文字起こし：会議や打ち合わせを録音し、AIが自動で文字起こし・クリーニング・要約
  • 🗄️ 保管庫：テキストメモ・ファイル・Webデータ・録音データを一元管理
  • 🔍 照会室：保管庫のデータをもとにAIが質問に回答
  • 💬 相談室：複数のAIペルソナが多角的な視点から意見を提示
  • ⚙️ 設定：AIモード・ペルソナ・カテゴリの管理
✅ ローカルモード時、インターネット接続は初期セットアップ時のダウンロードのみに使用します。

2. AIモードについて

COVEはナビゲーションバーに現在のAIモードを常時表示します。機密情報を扱う前に必ず確認してください。

4. インストール手順

📋 はじめる前に ― コマンドラインツールについて

セットアップでは コマンドラインツール を使います。このページのコマンドをコピーして貼り付けてEnterを押すだけでOKです。

■ コマンドラインツールの開き方

  1. 画面左下の スタートボタンを右クリック
  2. メニューから 「PowerShell」 または 「ターミナル」 をクリック
  3. 黒い画面（またはウィンドウ）が開いたら準備完了
  1. キーボードの Cmd（⌘）+ Space を押してSpotlight検索を開く
  2. 「Terminal」 と入力してEnterを押す
  3. ウィンドウが開いたら準備完了

■ Step 1: uvのインストール

winget install --id=astral-sh.uv -e
curl -LsSf https://astral.sh/uv/install.sh | sh

インストール後、Terminalを再起動してください。

■ Step 2: Ollamaのインストール

Ollamaはローカルでアシスタントを動かすためのソフトウェアです。

  1. https://ollama.com/download にアクセス
  2. 「Download for Windows」をクリックしてダウンロード
  3. ダウンロードした .exe ファイルを実行してインストール
brew install ollama

Homebrewがない場合は https://brew.sh からインストールしてください。

✅ インストール後、ollama --version と打って、バージョン番号が表示されれば成功です。

■ Step 3: AIモデルのダウンロード

COVEで使用するAIモデルをダウンロードします。約17GBあるため、安定したインターネット環境で行ってください。

ollama pull gemma3:27b
⚠️ ダウンロードには30分〜2時間程度かかります。完了するまでウィンドウを閉じないでください。

■ Step 4: ffmpegのインストール

音声のノーマライズ（音量均一化）・トリミング機能に使用します。

winget install ffmpeg
brew install ffmpeg

■ Step 5: COVEのセットアップ

cd cove-by-aura
uv sync
ℹ️「uv sync」は初回のみ必要です。以後は起動コマンドのみでOKです。

5. 起動・終了方法

■ 起動

cd cove-by-aura
uv run python main.py

起動後、ブラウザが自動で開きます。開かない場合は http://127.0.0.1:5001 にアクセスしてください。

⚠️ コマンドラインウィンドウを閉じないでください。閉じるとCOVEが終了します。

■ 終了

画面右上の「⏻ 終了」ボタンをクリックします。

⚠️ ブラウザのタブを閉じるだけではCOVEは終了しません。必ず「終了」ボタンを使用してください。

6. 画面の使い方

■ 🎙️ 録音画面

  1. 「録音開始」ボタンを押して録音を開始
  2. 録音が終わったら「録音停止」ボタンを押す
  3. 自動で文字起こし → クリーニング → 要約が実行される
  4. 完了後、3つのタブで各段階の結果を確認できる

■ 結果の3タブ表示

タブ内容
📝 文字起こしfaster-whisperによる生の文字起こし結果
🧹 クリーニング済みフィラー・重複除去・改行整形済みのテキスト
✨ AI要約発言録と概要まとめの2部構成
ℹ️ 要約の内容に不明な点があれば「クリーニング済み」タブで原文を確認できます。

■ 手動再試行

録音履歴の各カードから個別に再処理できます。

  • 🔄 文字起こし：文字起こしからやり直し
  • 🧹 再クリーニング：クリーニングのみ再実行
  • 🔄 再要約：要約のみ再実行

■ 手動トリミング

  1. 録音一覧の「✂️ トリミング」ボタンをクリック
  2. 波形を見ながら開始・終了スライダーで範囲を指定
  3. 「✂️ トリミング実行」で上書き保存

■ 🗄️ 保管庫

  • テキストメモ：自由にメモを入力。「利用者情報」カテゴリは相談・照会で常に参照される
  • ファイル：PDFや文書ファイルをアップロードしてAIが要約
  • Webデータ：URLを入力して参考ページの内容を取得
  • 録音データ：録音済みの音声を追加してテキスト化

■ 🔍 照会室

保管庫に蓄積したデータをもとにAI（アーカイバー）が質問に回答します。過去の会議内容・メモ・ファイルを横断的に参照できます。

  1. カテゴリと参照するデータを選択
  2. 質問を入力して「照会する」を押す
  3. アーカイバーがデータをもとに回答
  4. フォローアップ質問で深掘りできる
ℹ️ 保管庫のデータが充実するほど、照会の精度が上がります。

■ 💬 相談室

  1. カテゴリとペルソナグループを選択
  2. 相談内容を入力して「相談する」を押す
  3. 各ペルソナが順番に意見を述べる（画面下部のパネルで確認）
  4. 採用したい回答の「採用する」ボタンを押す
  5. 「相談終了」で終了し、最終判断メモを入力して保存
ℹ️ ← → ボタンでペルソナの回答を切り替えられます。採用した回答に★評価をつけると履歴で管理できます。

■ ⚙️ 設定画面

AIモード設定

  • ローカル／クラウドをスイッチで切り替え、サービスをドロップダウンで選択
  • クラウドモード選択時はAPIキーを入力して保存
  • 現在のモードはナビゲーションバーに常時表示される

最大録音時間（Ollamaモード時）

  • PCのRAMに基づいて選択肢が自動生成される（30分単位・目安）
  • 長時間ほど要約に使用するメモリが増え、処理時間も長くなる

ペルソナ管理

  • ペルソナ名またはアイコンをクリック → 編集ポップアップが開く
  • アイコンは待機用・考え中用の2種類を設定可能
  • システムプロンプトを変更するとAIの回答スタイルが変わる
  • グループ管理でペルソナをグループ分けし、相談室で切り替えられる
🚫 デンジャーゾーン：録音・メモ・ファイル・Webデータ・相談履歴の全削除が可能です。取り消せないため慎重に操作してください。

8. トラブルシューティング

■ 起動できない

「uvが見つかりません」：Step 1のuvインストールを確認し、PowerShell/Terminalを再起動してください。

ブラウザが開かない：手動で http://127.0.0.1:5001 にアクセスしてください。

■ Ollamaが起動しない

  • タスクマネージャーで「ollama.exe」が動いているか確認
  • 動いていない場合はスタートメニューから「Ollama」を起動
  • または以下のコマンドで強制停止後に再起動
taskkill /F /IM ollama.exe
  • メニューバーにOllamaのアイコンがあるか確認
  • ない場合はApplicationsフォルダからOllamaを起動
  • またはTerminalで ollama serve を実行

■ AIモデルが見つからない

以下のコマンドでモデルをダウンロードしてください。

ollama pull gemma3:27b

VRAMが不足する場合は小さいモデルをお試しください。

ollama pull gemma3:12b

■ 要約に時間がかかる・処理が止まる

1時間以上の長い録音では要約処理に30分以上かかる場合があります。

  • 設定画面の「最大録音時間」を短く設定するとメモリ使用量が減り安定する
  • 処理中は他のアプリの起動・ファイル編集を控える
  • 処理が完全に止まった場合は以下で強制停止できる
ollama stop gemma3:27b
ℹ️ 処理中のOllamaの状態は ollama ps で確認できます。CONTEXTが8192の場合は設定画面で最大録音時間を確認してください。

■ ffmpegが見つからない（トリミング・ノーマライズエラー）

winget install ffmpeg
brew install ffmpeg

インストール後、COVEを再起動してください。

■ 録音できない（マイクが認識されない）

  • OSの設定でマイクへのアクセスを許可する
  • 録音画面を再読み込みする（Ctrl+R / Cmd+R）

■ Mac：システム音声が録音できない

  1. https://existential.audio/blackhole/ からBlackHoleをダウンロード・インストール
  2. COVEを再起動する

9. プライバシー・ライセンス

✅ ローカルモード（Ollama）時、COVEはすべての処理をPC上でローカルに実行します。入力した情報・録音データ・相談内容は外部サーバーに送信されません。
⚠️ クラウドモード（Groq・OpenAI・Gemini・Claude）使用時は、音声データとテキストデータが各サービスのサーバーに送信されます。各サービスのプライバシーポリシーに従って管理されます。機密情報を含む場合はローカルモードをご利用ください。

■ データの保存場所

  • データベース：cove-by-aura/app/cove.db
  • 録音ファイル：cove-by-aura/uploads/
  • 設定ファイル：cove-by-aura/.env
